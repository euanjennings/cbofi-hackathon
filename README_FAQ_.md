# Collaboration Sandpits FAQs

## Access & Memory

### Q1. Who will be able to access the Collaborative Data Sandpit?

Participants in the same project can access the collaborative data sandpit, providing a common workspace for collaboration. Users from other projects will not have the ability to view or access the collaborative data sandpits of projects they are not a part of.

### Q2. Can participants work together in real-time within the same notebook?

Yes, real-time collaboration (RTC) is supported, allowing participants to work in the same notebook simultaneously. This feature enables users to see others typing and working in real time. Participants have access to the same directory and can work on multiple notebooks concurrently, enhancing teamwork and productivity.

### Q3. How does the runtime environment function when participants are working together?

When participants are working in the same notebook, cells are executed in a queue, mimicking the behavior of a single-user environment. However, if participants are working on separate notebooks, they can run their cells in parallel, allowing for efficient multitasking and collaboration without interrupting each other's workflows.

### Q4. How is memory managed when multiple participants are collaborating within a notebook?

All participants share the same memory allocation provided to a notebook. This means that the memory resources used collectively by everyone working within the same collaborative environment.

### Q5. What type of access do participants in the same project have in the collaborative data sandpit?

Participants within the same project have both read and write access in the collaborative data sandpit. This enables them to not only view content and data but also make changes, additions, and deletions, facilitating a truly collaborative workspace.

### Q6. If I start or stop the server, does this affect everyone in the project?

Yes, starting or stopping the server affects all participants in the project. If you start the server, it becomes active for everyone involved, and similarly, stopping the server will halt operations for all users currently working within the environment.

### Q7. Where are my saved datasets and files stored within the collaborative data sandpit?

All files and datasets you save are stored within the collaborative data sandpit itself and are accessible exclusively to you and the other participants in your project. This ensures that your work remains available only to those who are authorized to view and edit within your collaborative workspace.
